1. ภาพรวมโครงการ (Project Overview)
ปัจจุบันการดึงข้อมูล Downtime % ของแต่ละเครื่องต้องทำทีละ Asset Code ผ่านหน้าเว็บ ซึ่งใช้เวลามากและเกิด human error ได้ง่าย
เครื่องมือนี้จะ automate การดึงข้อมูลจาก TERAHOP EAM API โดยอัตโนมัติ สำหรับหลาย Asset Code พร้อมกัน และ export ผลลัพธ์เป็น Excel รายวัน พร้อม Downtime % สำเร็จรูป
1.1 วัตถุประสงค์
ดึงข้อมูล Downtime รายวันของทุกเครื่องในครั้งเดียว
คำนวณ Downtime % โดยอัตโนมัติ ไม่ต้องคำนวณมือ
Export เป็น Excel พร้อมใช้งาน / วิเคราะห์ต่อได้ทันที
รองรับการ schedule รันอัตโนมัติทุกวัน (optional)
1.2 Scope
ระบบ: TERAHOP EAM — Device Utilization Report
URL: http://10.190.0.184:8080/eam/report_management/eqp_report/eqp_utilization_rate
Station: AA (ขยายได้ภายหลัง)
Output: Excel file รายวัน / รายสัปดาห์
2. สูตรการคำนวณ (Calculation Logic)
2.1 Total Available Time ต่อเครื่อง
เนื่องจาก API ไม่ได้ส่ง Total Available Time มาโดยตรง ต้องคำนวณจาก:
Total_i  = RunTime_i ÷ Utilization_iตัวอย่าง: RunTime = 158.806h, Utilization = 0.87944Total_i  = 158.806 ÷ 0.87944 = 180.58h
2.2 Downtime % ต่อเครื่อง (รายวัน)
Downtime%_i = (Downtime_i ÷ Total_i) × 100
2.3 Overall Downtime % (ทุกเครื่องรวมกัน)
Downtime%_overall = (Σ Downtime_i) ÷ (Σ Total_i) × 100หมายเหตุ: ใช้ weighted sum — ไม่ใช่ averageเครื่องที่ Utilization = 0% → exclude ออกจากการคำนวณ (หาร 0 ไม่ได้)

4. API Specification
4.1 สิ่งที่ต้องหาจาก DevTools ก่อน Implement
Programmer ต้องทำขั้นตอนนี้ก่อน เพื่อยืนยัน API contract:
เปิดหน้าเว็บ Device Utilization Report
กด F12 → tab Network
ใส่ Asset Code ใดก็ได้ + กำหนด date → กด Query
ดู request ที่ยิงออก → copy URL, Method, Headers, Request Body, Response JSON

6. Logic การทำงานหลัก (main.py)
1. โหลด config.json และ asset_list.txt2. For each asset_code in asset_list:      a. เรียก api_caller.get_daily_data(asset_code, start, end)      b. ได้ list ของ {date, runTime, downtime, utilization}      c. เรียก calculator.compute(records)         └→ total_i = runTime / utilization         └→ downtime_pct = downtime / total_i × 100         └→ idle_h = total_i − runTime − downtime         └→ idle_pct = idle_h / total_i × 1003. รวมทุก asset → overall summary row4. เรียก exporter.to_excel(all_records, filename)5. Print ‘Done: output/downtime_YYYYMMDD.xlsx’

9. Error Handling
HTTP 401 / 403 → print ‘Token expired — update config.json’ แล้วหยุด
HTTP 500 → retry 3 ครั้ง, ถ้ายังผิด → skip asset + log
Utilization = 0 → skip การคำนวณ Total (หาร 0) + เขียน N/A ใน Excel
Network timeout → retry ด้วย exponential backoff (1s, 2s, 4s)
ทุก error บันทึกลง error_log.txt พร้อม timestamp
